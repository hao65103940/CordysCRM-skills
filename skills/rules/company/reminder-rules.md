# 线索跟进提醒规则

## 📋 规则说明

当用户查询**自己名下的线索**时，需要检查以下两个时间阈值，并给出友好提醒：

### 规则 1：领取时间即将超期（90 天）

**条件：** `collectionTime` 距离现在 ≥ 87 天（即剩余 ≤ 3 天）

**提醒文案：**
```
⚠️ 以下线索领取即将超过 90 天，请尽快跟进或处理：

| 线索名称 | 领取时间 | 已领取天数 | 剩余天数 |
|---------|---------|-----------|---------|
| XXX 公司 | 2026-01-15 | 88 天 | 2 天 |
```

**超期文案（≥ 90 天）：**
```
🚨 以下线索领取已超过 90 天，可能面临回收风险：

| 线索名称 | 领取时间 | 超期天数 |
|---------|---------|---------|
| XXX 公司 | 2026-01-14 | 超期 1 天 |
```

---

### 规则 2：跟进时间超时（30 天）

**条件：** `followTime` 距离现在 ≥ 27 天（即剩余 ≤ 3 天）或为 null

**提醒文案：**
```
⚠️ 以下线索超过 27 天未跟进，建议尽快联系客户：

| 线索名称 | 最后跟进时间 | 未跟进天数 |
|---------|------------|-----------|
| XXX 公司 | 2026-03-17 | 27 天 |
```

**严重超时文案（≥ 30 天）：**
```
🚨 以下线索超过 30 天未跟进，存在流失风险：

| 线索名称 | 最后跟进时间 | 未跟进天数 |
|---------|------------|-----------|
| XXX 公司 | 2026-03-14 | 30 天 |
```

---

## 🔧 执行流程

### 步骤 1：查询线索时自动检查

当用户查询线索（`cordys crm page lead`）时：
1. 检查查询条件是否包含 `owner` 或 `follower`（即查询自己名下的线索）
2. 如果是，自动检查每条线索的 `collectionTime` 和 `followTime`

### 步骤 2：计算时间差

```python
import datetime

now = datetime.datetime.now()
collection_date = datetime.datetime.fromtimestamp(collectionTime / 1000)
follow_date = datetime.datetime.fromtimestamp(followTime / 1000) if followTime else None

days_since_collection = (now - collection_date).days
days_since_follow = (now - follow_date).days if follow_date else 999
```

### 步骤 3：判断是否需要提醒

```python
warnings = []

# 检查领取时间
if days_since_collection >= 90:
    warnings.append(('expired_collection', days_since_collection - 90))
elif days_since_collection >= 87:
    warnings.append(('warning_collection', 90 - days_since_collection))

# 检查跟进时间
if followTime is None or days_since_follow >= 30:
    warnings.append(('expired_follow', days_since_follow if followTime else '从未跟进'))
elif days_since_follow >= 27:
    warnings.append(('warning_follow', 30 - days_since_follow))
```

### 步骤 4：输出提醒

**优先级：** 超期 > 预警 > 正常

**输出位置：** 在查询结果**之前**先输出提醒

## 🎯 触发场景

| 用户指令 | 是否触发检查 | 说明 |
|---------|------------|------|
| "查一下我的线索" | ✅ 触发 | 查询自己名下的线索 |
| "查东区的所有线索" | ❌ 不触发 | 查询区域线索，不一定是自己的 |
| "查我名下超过 30 天没跟进的线索" | ✅ 触发 | 明确查询自己的线索 |
| "查张三的线索" | ❌ 不触发 | 查询他人的线索 |

---

## 📌 注意事项

1. **仅对自己名下的线索生效**：通过 `owner` 或 `follower` 字段判断
2. **时间计算基准**：使用当前系统时间（Asia/Shanghai 时区）
3. **null 值处理**：`followTime` 为 null 时，视为"从未跟进"，直接触发严重超时提醒
4. **提醒优先级**：超期提醒 > 预警提醒 > 正常结果
5. **友好文案**：使用 emoji 和表格，让提醒清晰易读
